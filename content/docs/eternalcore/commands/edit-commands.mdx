## ğŸ”§ Example configuration

```yaml
commands:
  eternalcore:
    name: "eternal-core"
    enabled: true
    aliases:
      - "eternal"
    permissions:
      - "eternalcore.eternalcore"
    subCommands:
      reload:
        name: "reload"
        enabled: true
        aliases:
          - "rl"
        permissions:
          - "eternalcore.reload"
```

## âœ… Result of this configuration

* `/eternalcore` becomes `/eternal-core`
* New alias `/eternal` is added
* The `/eternal-core reload` command can now also be run using `/eternal-core rl`
* Both commands require custom permissions
